# Portfolio with Real-time Chat & Admin Panel This Next.js portfolio includes: - Real-time chat room - Contact form with Firebase integration - Admin panel for managing messages and contacts ## Setup Instructions 1. **Install Dependencies** ```bash npm install firebase ``` 2. **Firebase Configuration** - The Firebase config is already set up in `lib/firebase.js` - Make sure your Firebase project has Firestore enabled 3. **Firebase Security Rules** Add these rules to your Firestore: ```javascript rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /chatMessages/{document} { allow read, write: if true; } match /contactForms/{document} { allow read, write: if true; } } } ``` 4. **Admin Access** - Default admin password: `admin123` - Change this in `components/AdminPanel.jsx` line 15 - For production, implement proper authentication ## Features ### Chat Room (`/chat-room`) - Real-time messaging - User identification - Message timestamps - Responsive design ### Contact Form (`/contact`) - Form validation - Firebase integration - Success/error feedback - Professional design ### Admin Panel (`/dashboard`) - Password protection - View all chat messages - Manage contact form submissions - Mark contacts as read/unread - Delete messages and contacts - Real-time statistics ## File Structure ``` components/ ├── ChatRoom.jsx       # Real-time chat component ├── ContactForm.jsx    # Contact form with Firebase ├── AdminPanel.jsx     # Admin dashboard └── Navigation.jsx     # Navigation component lib/ └── firebase.js        # Firebase configuration app/ ├── chat-room/page.js  # Chat room page ├── contact/page.js    # Contact page └── dashboard/page.js  # Admin dashboard page ``` ## Usage 1. **Chat Room**: Users can join with a name and chat in real-time 2. **Contact Form**: Visitors can submit inquiries that go to Firebase 3. **Admin Panel**: Access with password to manage all data ## Security Notes - Change the admin password in production - Implement proper authentication for admin access - Review Firebase security rules for your use case - Consider rate limiting for chat messages
