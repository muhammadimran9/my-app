# Theme System Usage Guide

## Overview
The theme system provides a complete Light/Dark mode toggle for your "Spot For You" website with instant switching, localStorage persistence, and system preference detection.

## Components Available

### 1. ThemeProvider
Wrap your app with this provider (already done in layout.js):
```jsx
import { ThemeProvider } from '../contexts/ThemeContext';

<ThemeProvider>
  {/* Your app content */}
</ThemeProvider>
```

### 2. useTheme Hook
Access theme state in any component:
```jsx
import { useTheme } from '../contexts/ThemeContext';

function MyComponent() {
  const { theme, toggleTheme, isDark } = useTheme();
  
  return (
    <div>
      <p>Current theme: {theme}</p>
      <button onClick={toggleTheme}>
        Switch to {isDark ? 'light' : 'dark'} mode
      </button>
    </div>
  );
}
```

### 3. ThemeToggle Component
Ready-to-use toggle button:
```jsx
import ThemeToggle from '../components/ThemeToggle';

<ThemeToggle className="my-custom-class" />
```

### 4. ThemedButton Component
Theme-aware button with variants:
```jsx
import ThemedButton from '../components/ThemedButton';

<ThemedButton variant="primary" size="md">
  Click me
</ThemedButton>

// Variants: primary, secondary, outline, ghost
// Sizes: sm, md, lg
```

### 5. ThemedCard Component
Theme-aware card container:
```jsx
import ThemedCard from '../components/ThemedCard';

<ThemedCard hover={true} padding="lg">
  <h3>Card Title</h3>
  <p>Card content</p>
</ThemedCard>
```

## CSS Variables Available

Use these CSS variables in your components for theme consistency:

### Colors
- `var(--background)` - Main background color
- `var(--foreground)` - Main text color
- `var(--card-bg)` - Card background
- `var(--card-border)` - Card border color
- `var(--button-bg)` - Button background
- `var(--button-hover)` - Button hover state
- `var(--button-text)` - Button text color
- `var(--link-color)` - Link color
- `var(--link-hover)` - Link hover color
- `var(--border-color)` - General border color
- `var(--input-bg)` - Input background
- `var(--input-border)` - Input border
- `var(--text-muted)` - Muted text color
- `var(--text-secondary)` - Secondary text color
- `var(--shadow)` - Box shadow color

### CSS Classes Available
- `.theme-card` - Card styling
- `.theme-button` - Button styling
- `.theme-link` - Link styling
- `.theme-input` - Input styling
- `.theme-text-muted` - Muted text
- `.theme-text-secondary` - Secondary text
- `.theme-border` - Border styling
- `.theme-shadow` - Shadow styling

## Example Component with Theme

```jsx
'use client';

import { useTheme } from '../contexts/ThemeContext';
import ThemedCard from '../components/ThemedCard';
import ThemedButton from '../components/ThemedButton';

export default function MyPage() {
  const { isDark } = useTheme();
  
  return (
    <div style={{ background: 'var(--background)', minHeight: '100vh' }}>
      <div className="container mx-auto p-4">
        <h1 style={{ color: 'var(--foreground)' }}>
          Welcome to {isDark ? 'Dark' : 'Light'} Mode
        </h1>
        
        <ThemedCard hover={true} className="mb-4">
          <h2 style={{ color: 'var(--foreground)' }}>Card Title</h2>
          <p style={{ color: 'var(--text-secondary)' }}>
            This card automatically adapts to the current theme.
          </p>
          <ThemedButton variant="primary">
            Primary Button
          </ThemedButton>
        </ThemedCard>
        
        <div className="theme-card p-4 rounded-lg">
          <p style={{ color: 'var(--text-muted)' }}>
            Using theme-card class for consistent styling.
          </p>
        </div>
      </div>
    </div>
  );
}
```

## Features Implemented

✅ **Instant Theme Switching** - No page refresh needed
✅ **localStorage Persistence** - Theme persists after browser refresh
✅ **System Preference Detection** - Follows user's OS theme preference
✅ **Smooth Transitions** - All elements transition smoothly between themes
✅ **Global Coverage** - Works on all pages and components
✅ **No Mixed Colors** - Prevents dark text on dark backgrounds
✅ **Consistent Styling** - All components follow the same theme rules

## Theme Rules Applied

### Light Mode
- Background: White/Light colors
- Text: Black/Dark colors
- Cards: Light backgrounds with subtle borders
- Buttons/Links: Blue theme with proper contrast

### Dark Mode
- Background: Black/Dark colors
- Text: White/Light colors
- Cards: Dark backgrounds with subtle borders
- Buttons/Links: Blue theme optimized for dark backgrounds
